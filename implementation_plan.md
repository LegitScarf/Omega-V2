# Implementation Plan: Evolving Omega from Deterministic Pipelines to Agentic Reasoning (Omega V3)

## Goal Description
Omega V2 utilizes a deterministic router that classifies a query's intent type (e.g., `descriptive`, `trend`, `regression`) and executes a hardcoded template. While fast, this structure fails on complex, multi-step analytical requests (such as the EV Adoption dataset's questions). To make Omega a robust alternative to Excel, SQL, Python, and Power BI, we must transition to an agentic reasoning platform (**Omega V3**).

Omega V3 will use an autonomous **ReAct (Reasoning + Acting) Agent** equipped with a local **Python Code Execution Sandbox**. Instead of routing queries to predefined statistical solvers, the LLM will reason about the data schema, write custom Python code, execute it locally, capture output, self-correct if exceptions occur, and serialize results into the format expected by the Streamlit dashboard UI.

---

## User Review Required

> [!IMPORTANT]
> **Code Sandbox Safety & Local Execution**
> All code generated by the LLM is executed locally on the user's host environment where the Streamlit app runs. The sandbox will capture `stdout` and `stderr` and support key libraries (`pandas`, `numpy`, `scipy`, `statsmodels`, `plotly`). No external cloud/internet APIs are invoked during sandbox execution, preserving local data privacy.

> [!WARNING]
> **Performance Impact**
> Evolving to a ReAct loop with potential self-correction cycles may increase query response times from sub-1 second to **1.5 to 3.0 seconds**. We believe the trade-off for high accuracy and reasoning capacity is highly justified.

### 2. ReAct Agent Loop & Self-Correction

#### [MODIFY] [crew.py](file:///c:/Users/KIIT/Desktop/Omega_Streamlit/src/crew.py)
We will replace `run_omega` and the intent parser with an autonomous agent loop:
- **Planner Phase:** The LLM receives the user query, dataframe columns, data types, and a 5-row sample.
- **Code Generation:** The LLM generates a complete Python script to load the data, perform calculations, and save output metrics/plots.
- **Execution & Evaluation:** If the script encounters an error (e.g., `KeyError`, type mismatch), the agent catches the traceback and feeds it back to the LLM. The LLM then has up to 3 self-correction attempts to repair the script.
- **JSON Serialization:** The generated script will write standard outputs (`eda_result.json`, `query_result.json`, `chart.json`, `hypothesis_test.json`, `prediction.json`, `insight.json`) to the output folder.

#### Expected JSON Artifact Structure to support existing UI:
- **`chart.json`**:
  ```json
  {
    "status": "success",
    "chart_generated": true,
    "plotly_spec": "<Plotly figure exported as JSON via fig.to_json()>",
    "chart_type": "scatter/bar/line/etc",
    "chart_title": "Title"
  }
  ```
- **`query_result.json`**:
  ```json
  {
    "status": "success",
    "result_rows": [{"col1": val1, "col2": val2}],
    "row_count": 100,
    "truncated": false
  }
  ```
- **`prediction.json`**: (Depending on query type)
  ```json
  {
    "status": "success/regression/classification/clustering",
    "target_column": "target",
    "features": ["feat1", "feat2"],
    "model_metrics": {"R2/Accuracy": 0.85}
  }
  ```
